ExFAT Shell

Problem: File systems are opaque. Users need a way to interact with their files securely.

I built a custom shell in C for the ExFAT file system with support for common UNIX-style commands and executable handling. The project pushed me to think carefully about parsing, process behavior, and predictable system interaction.

COMP 3430 · Operating Systems

Cooperative Threads Library

Problem: Concurrency is hard to reason about. Programs stall, deadlock, or run unpredictably when threads compete.

I built a thread library in C for a single-core cooperative system, modeled loosely on pthreads. It includes mutex support and multiple scheduling policies, which made it a solid way to learn where concurrency becomes hard to control.

  • Supports up to 1,000 threads per process
  • Enables preemptive scheduling of threads
  • Mutex support included with the library
  • 4 scheduling policies: FIFO, 2-Thread, Round Robin, and MLFQ

Result: The project made scheduling and synchronization feel much more concrete. It also forced me to be precise about edge cases instead of hand-waving them away.

Published under a restrictive license. Source code available upon request.
